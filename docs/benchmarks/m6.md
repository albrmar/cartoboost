# M6 Forecasting Metrics

This page documents the CartoBoost M6 metric helper contract. It is not a new
benchmark claim. Public benchmark claims still require a real run with the
command, data source, split boundaries, model roster, comparable settings,
metric table, timings, artifacts, and interpretation recorded.

## Metric Helpers

The M6 helper combines:

- pinball loss for quantile return forecasts;
- rank probability score for ordered rank-distribution forecasts;
- an equal-weight combined score: `0.5 * pinball + 0.5 * rps`.

Inputs must be finite and aligned. Probability vectors must be non-negative and
sum to one. Observed ranks are zero-based category indexes.

```python
from pathlib import Path
import importlib.util

path = Path("python/cartoboost/metrics/m6.py")
spec = importlib.util.spec_from_file_location("cartoboost_metrics_m6", path)
m6 = importlib.util.module_from_spec(spec)
spec.loader.exec_module(m6)

summary = m6.evaluate_m6_metrics(
    actual_returns=[10.0, 20.0],
    quantile_predictions=[11.0, 18.0],
    quantile=0.5,
    rank_probabilities=[0.2, 0.3, 0.5],
    observed_rank=2,
)
```

The helper returns `pinball_loss`, `rank_probability_score`, and
`combined_score`. It does not load market data, construct benchmark splits, or
make portfolio-performance claims.

## Portfolio Construction

The Rust finance helper provides deterministic long/short portfolio
construction from asset return signals. Assets are ranked by expected return
descending, ties are broken by asset ID, selected long positions receive equal
positive weights, selected short positions receive equal negative weights, and
the final position list is sorted by asset ID for reproducible output.

## Current Evidence Boundary

The maintained forecasting benchmark index records an existing M6 artifact at:

`docs/assets/nyc_taxi_benchmarks/forecasting_m6_full.json`

Use that artifact only with its recorded command and report context. Adding or
changing M6 benchmark claims requires a refreshed writeup with exact command,
data source, sample size, task and split definitions, model roster, comparable
settings, metric breakdown tables, timing breakdown, artifact paths, plots,
winner or tie interpretation, limitations, and why the result is structurally
meaningful.
